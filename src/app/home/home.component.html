What this project demonstrates

Angular UI Architecture

Feature-oriented UI structure, typed models, service-based API communication, Angular Material components, and route-based navigation.

Backend Integration Design

The frontend consumes a Spring Boot backend that normalizes external API data, hides third-party credentials, and presents cleaner contracts to the client.

AWS Hosting

The frontend is deployed with AWS Amplify, and the backend is deployed separately behind its own HTTPS endpoint. The application is structured to reflect real deployment concerns.

AWS deployment details

Beyond the frontend hosting layer, this project also demonstrates how I deploy and configure a backend service in AWS using managed infrastructure, HTTPS, domain routing, and secure configuration practices.

Route 53

Routes traffic for my custom domain and subdomains, including the frontend site and backend API endpoint.

AWS Amplify

Hosts and deploys the Angular frontend from source control as the public-facing portfolio experience.

ACM + HTTPS

Uses AWS Certificate Manager to support HTTPS for the backend domain and demonstrate secure production-style traffic handling.

Elastic Beanstalk + EC2

Deploys the Spring Boot backend through Elastic Beanstalk, with the application running on underlying EC2 infrastructure managed by the platform.

Load Balancer

Uses an Application Load Balancer to terminate TLS and route requests to the backend service.

Secrets and Configuration

Keeps sensitive backend configuration out of the frontend and out of source control, reflecting secure environment-based deployment practices.
